# GitHub Handoff

This document covers the safe path for putting MeldSync on GitHub.

## Before Creating the Repo

Confirm these files are ignored:

- `melds_report__*.csv`
- `*.private.csv`
- `private/`
- `meldsync-backup-*.json`
- `.env`
- `.env.*`

The copied real Property Meld CSV should remain local only.

## Local Git Setup

After the local repo is initialized and committed, create an empty GitHub repo with no README/license/gitignore generated by GitHub. The repo now exists at `https://github.com/hypnoticdata777/m3ldSync.git`.

Run:

```powershell
git remote add origin https://github.com/hypnoticdata777/m3ldSync.git
git branch -M main
git push -u origin main
```

If using SSH:

```powershell
git remote add origin git@github.com:hypnoticdata777/m3ldSync.git
git branch -M main
git push -u origin main
```

## Codex Safe Directory Note

Codex may need a per-command safe-directory override because the Desktop folder is owned by your Windows user while Codex runs as a sandbox user.

Codex can use:

```powershell
git -c safe.directory='C:/Users/hypno/OneDrive/Desktop/m3ldSync' status
```

When you run Git directly in your own terminal, you may not need this override.

## Pre-Push Safety Check

Run:

```powershell
git status --short --ignored
```

Confirm:

- The real `melds_report__*.csv` file is ignored.
- No backup JSON files are staged.
- No `.env` files are staged.
- Only source, tests, and documentation are staged/committed.
